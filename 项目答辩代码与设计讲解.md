# 智能冰箱视觉管理系统：答辩代码与设计讲解

## 一、项目整体设计

本项目名称为“智能冰箱视觉管理系统”，不是一个单纯的 YOLO 目标检测 Demo，而是一套完整的图像处理应用系统。

系统输入是一张冰箱内部图片，之后依次完成：

- 图像预处理
- 食材目标检测
- 数量统计
- 面积估算
- 空间定位
- 热力图分析
- 补货提醒
- 空间整理建议
- Web 页面展示
- CSV / PNG 结果导出

整体流程如下：

```text
上传冰箱图片
→ 图像预处理
→ YOLOv8 食材检测
→ 检测结果结构化
→ 面积估算
→ 空间位置分析
→ 热力图生成
→ 库存统计
→ 补货提醒 / 整理建议
→ Web 页面展示 / CSV 和 PNG 导出
```

项目对应课程要求：

```text
难度 1.0：基础图像处理功能
难度 1.3：目标检测与定位
难度 1.5：结合实际场景的应用功能
```

## 三、界面主程序设计

主界面代码位于：

```text
app/ui/streamlit_app.py
```

它负责整个系统的前端交互和流程调度。

主要功能：

1. 设置 Streamlit 页面。
2. 在左侧 sidebar 提供参数控制。
3. 上传冰箱图片。
4. 根据参数执行图像预处理。
5. 加载 YOLO 模型。
6. 运行食材检测。
7. 生成面积、位置、库存、热力图等结果。
8. 根据顶部导航栏展示不同页面。
9. 提供 CSV 和 PNG 导出。

左侧参数包括：

```text
预处理参数：
灰度化、CLAHE 对比度增强、去噪、Canny 边缘检测、形态学处理

检测参数：
模型选择、置信度阈值、NMS IoU 阈值、是否使用原图检测

结果展示：
是否显示灰度图、去噪图、边缘图、形态学图
```

设计亮点：

系统不是一次性跑死流程，而是把图像处理流程做成可调参数。答辩时可以现场演示不同预处理开关对结果的影响。

## 四、图像预处理模块

代码位于：

```text
app/core/image_preprocess.py
```

主要函数：

```python
load_image()
convert_gray()
enhance_contrast()
remove_noise()
edge_detection()
morphological_process()
preprocess_pipeline()
```

### 1. `load_image()`

负责读取图片。

支持两种输入：

- 本地图片路径
- Streamlit 上传文件

读取后返回 OpenCV 使用的 BGR 格式图像。

### 2. `convert_gray()`

将 BGR 图像转换成灰度图。

答辩可讲公式：

```text
Gray = 0.299R + 0.587G + 0.114B
```

### 3. `enhance_contrast()`

使用 CLAHE 做对比度增强。

设计细节：

- 如果输入是灰度图，直接对灰度图做 CLAHE。
- 如果输入是彩色图，先转换到 LAB 色彩空间。
- 只增强 L 亮度通道。
- 再合并通道并转换回 BGR。

这样做的好处：

```text
增强暗区细节，同时减少颜色失真。
```

### 4. `remove_noise()`

先做高斯滤波，再做中值滤波。

```text
高斯滤波：平滑高斯噪声。
中值滤波：去除椒盐噪声，同时更好保留边缘。
```

### 5. `edge_detection()`

使用 Canny 边缘检测。

Canny 主要步骤：

```text
高斯平滑
→ 梯度计算
→ 非极大值抑制
→ 双阈值连接
```

### 6. `morphological_process()`

执行形态学开闭运算。

```text
开运算：先腐蚀后膨胀，用于去除小噪点。
闭运算：先膨胀后腐蚀，用于填补小空洞。
```

### 7. `preprocess_pipeline()`

将上述步骤组合成一条可配置管线。

每一步都可以通过参数开关控制：

```python
use_gray
use_contrast
use_denoise
use_edge
use_morphology
```

这个模块对应课程要求中的基础图像处理功能。

## 六、面积估算模块

代码位于：

```text
app/core/area_estimator.py
```

核心思想：

使用检测框面积近似食材的可见投影面积。

核心公式：

```text
pixel_area = (x2 - x1) × (y2 - y1)
estimated_area_mm2 = pixel_area × mm2_per_pixel
```

默认换算系数：

```python
DEFAULT_PIXEL_AREA_MM2 = 4.0
```

即：

```text
1 pixel ≈ 4.0 mm²
```

### 参考物标定设计

模块预留了参考物标定逻辑：

```python
REFERENCE_COIN_CLASS = "reference_coin"
REFERENCE_COIN_DIAMETER_MM = 25.0
REFERENCE_COIN_AREA_MM2 = 490.87
```

如果检测到参考硬币，可以使用：

```text
mm2_per_pixel = reference_area_mm2 / reference_pixel_area
```

如果没有检测到参考物，则使用默认系数。

### 半个/部分食材判断

系统会计算同一类别中每个目标的面积比例。

判断规则：

```text
某目标面积 / 该类最大面积 < 0.5
→ 标记为“半个/部分食材”

否则
→ 标记为“完整食材”
```

### 答辩说明

当前面积估算不是精确三维体积测量，而是基于二维图像的可见投影面积，用于相对比较和管理辅助。

## 七、空间定位模块

代码位于：

```text
app/core/position_analyzer.py
```

区域配置文件：

```text
config/fridge_regions.yaml
```

冰箱被划分成 3×3 九宫格。

垂直方向：

```text
上层：0.00 - 0.33
中层：0.33 - 0.66
下层：0.66 - 1.00
```

水平方向：

```text
左侧：0.00 - 0.33
中间：0.33 - 0.66
右侧：0.66 - 1.00
```

核心函数：

```python
get_food_position()
analyze_all_positions()
count_region_occupancy()
```

定位方法：

```text
ratio_x = center_x / image_width
ratio_y = center_y / image_height
```

示例：

```text
ratio_y = 0.2 → 上层
ratio_x = 0.7 → 右侧
最终位置 = 上层右侧
```

设计优点：

```text
与图像分辨率无关。
无论图片是 640×480 还是 1920×1080，都可以统一映射到冰箱区域。
```

## 八、热力图模块

代码位于：

```text
app/core/heatmap_generator.py
```

核心函数：

```python
generate_heatmap_overlay()
build_region_heatmap_summary()
```

热力图生成过程：

```text
1. 创建一张与原图同尺寸的空白 heatmap_canvas。
2. 对每个检测中心点画一个圆，默认半径 radius = 60。
3. 对 heatmap_canvas 做高斯模糊，sigmaX = 25，sigmaY = 25。
4. 归一化到 0-255。
5. 使用 OpenCV 的 COLORMAP_JET 进行伪彩色映射。
6. 与原图加权叠加，alpha = 0.45。
```

关键 OpenCV 函数：

```python
cv2.circle()
cv2.GaussianBlur()
cv2.normalize()
cv2.applyColorMap()
cv2.addWeighted()
```

答辩解释：

```text
红色区域表示食材集中，蓝色或绿色区域表示相对稀疏。
热力图可以帮助用户快速判断冰箱哪个区域更拥挤。
```

## 九、库存管理和应用建议模块

代码位于：

```text
app/core/inventory_manager.py
```

这是项目从“检测 Demo”升级为“实际应用系统”的关键模块。

主要函数：

```python
build_food_summary()
build_position_area_summary()
build_restock_suggestions()
build_space_advice()
```

### 1. `build_food_summary()`

生成食材库存汇总。

逻辑：

```text
按 class_name 分组
统计 count
合并每类食材 total_area_mm2
```

输出内容：

```text
食材类别
数量
总面积
```

### 2. `build_position_area_summary()`

把位置分析结果和面积估算结果合并。

输出内容包括：

```text
食材类别
位置
面积
完整/部分状态
置信度
中心点
```

### 3. `build_restock_suggestions()`

生成补货建议。

默认目标库存：

```text
apple: 2
banana: 2
orange: 2
tomato: 2
cucumber: 1
potato: 2
onion: 2
carrot: 2
green_pepper: 1
egg: 4
milk: 1
bread: 1
```

规则：

```text
gap = target_count - current_count

gap <= 0：
不提醒

gap > 0：
生成补货建议

current_count == 0：
高优先级

current_count > 0 但不足：
中优先级
```

### 4. `build_space_advice()`

生成空间整理建议。

规则包括：

```text
区域拥挤：
某区域检测到的食材数量 >= 3

面积集中：
某区域面积占总面积比例 >= 40%

可利用空位：
某些区域食材数量 <= 1
```

如果没有明显问题，则输出：

```text
当前各区域分布相对均衡，暂无明显拥挤位置。
```

## 十四、老师可能追问的问题

### 问题 1：为什么这是 App，不是脚本？

回答：

```text
本项目使用 Streamlit 构建了可交互 Web 应用，支持图片上传、参数调节、模型推理、结果展示和数据导出。用户可以通过界面完成完整操作，所以它是一个图像处理应用系统，而不是单独脚本。
```

### 问题 2：面积估算准不准？

回答：

```text
当前面积估算是基于检测框的二维投影面积，主要用于相对比较和管理辅助。系统预留了参考物标定逻辑，如果图片中加入硬币等已知尺寸物体，可以自动计算像素到真实面积的换算系数，提高精度。
```

### 问题 3：为什么要做热力图？

回答：

```text
检测框只能告诉用户每个食材的位置，但热力图能直观展示整体空间分布。红色区域表示食材集中，可以辅助判断冰箱哪些区域拥挤，从而生成整理建议。
```

### 问题 4：预处理会不会影响 YOLO 检测？

回答：

```text
会，所以系统提供了“检测时使用原图”选项。预处理结果主要用于展示数字图像处理过程，也可以作为检测输入进行对比。实际检测中通常原图更适合深度学习模型，预处理则用于课程功能展示和辅助分析。
```

### 问题 5：为什么用 3×3 九宫格？

回答：

```text
3×3 九宫格可以用较简单的规则覆盖冰箱的主要空间位置，包括上中下层和左中右区域。它既容易解释，也能满足空间整理建议的需求。后续如果需要更细粒度的位置分析，可以扩展区域配置。
```

### 问题 6：后续还能怎么扩展？

回答：

```text
后续可以加入摄像头实时检测、多帧视频分析、过期时间管理、实例分割、深度估计和嵌入式部署，使系统更接近真实智能冰箱终端。
```
