# 期末答辩说明

## 项目名称

智能冰箱视觉管理系统

## 项目类型

基于数字图像处理与目标检测的实际应用型课程项目

## 场景说明

本项目聚焦家庭冰箱食材管理场景，解决以下实际问题：

1. 用户不知道冰箱里还有哪些食材。
2. 同类食材数量不清楚，容易重复购买。
3. 冰箱内部摆放杂乱，不容易快速找到目标。
4. 用户缺少直观的空间利用信息，无法判断哪些区域最拥挤。

## 对应课程要求

### 1. 基础要求

项目已交付一套完整的图像处理系统 App，具有图像输入、处理、分析和结果展示能力。

### 2. 难度系数 1.3

项目实现了目标检测与定位功能，使用 YOLOv8 对冰箱中的食材进行检测，并输出类别、位置和数量统计。

### 3. 难度系数 1.5

项目在目标检测基础上增加了明确实际应用场景“智能冰箱管理”，并完成以下应用功能：

1. 库存统计与补货提醒。
2. 空间位置分析与整理建议。
3. 热力图辅助展示冰箱拥挤区域。

## 系统主要模块

1. 图像预处理模块：灰度化、增强、去噪、边缘检测、形态学处理。
2. 食材检测模块：YOLOv8 检测与定位。
3. 面积估算模块：估算食材投影面积。
4. 空间分析模块：判断食材所在冰箱区域。
5. 管理建议模块：输出补货提醒和空间整理建议。
6. 可视化展示模块：表格、标注图、热力图、导出结果。

## 答辩时推荐讲法

### 开场

“我做的是一个面向家庭冰箱管理的视觉系统，不只是识别图像里有什么，而是希望把识别结果变成可用的管理信息。”

### 中段

“系统先完成基础图像处理，再用 YOLOv8 进行食材检测与定位，然后对检测结果做数量统计、面积估算、区域分析和热力图生成。”

### 强调应用

“为了满足实际应用要求，我在检测结果基础上增加了两个管理功能：一是补货提醒，二是空间整理建议。这样系统就能从单纯识别升级为冰箱管理辅助工具。”

### 收束

“因此，这个项目既满足了数字图像处理基础要求，也满足了目标检测增强要求，并且具备明确场景和完整应用闭环。”

## 可以直接展示的功能点

1. 上传冰箱图片。
2. 展示预处理结果。
3. 展示 YOLO 检测框和类别。
4. 展示数量统计和面积估算表。
5. 展示区域位置与热力图。
6. 展示补货提醒。
7. 展示空间整理建议。
8. 导出结果图和 CSV。

## 老师如果问“为什么这是 App”

可以回答：

“这里的 App 指的是可交互的软件应用系统，不限定一定要是手机原生 App。我的系统使用 Streamlit 构建交互式界面，能够上传图片、运行分析、展示结果并导出数据，已经满足课程对图像处理系统或 App 的交付要求。”

## 老师如果问“后续还能做什么”

可以回答：

1. 增加摄像头实时检测。
2. 增加过期时间管理。
3. 增加菜谱推荐。
4. 增加实例分割和更精细面积估算。
5. 增加硬件联动，扩展为真正的智能冰箱终端。
